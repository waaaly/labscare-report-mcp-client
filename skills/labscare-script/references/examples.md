## 033水样采样记录表

- 返回根：`outputData` 数组
- 主过滤：`sample.tempId === '1143311212931801601'`
- 主分组：按 4 个采样人员签名 URL 组合分组
- 关键写法：
  - `getForm(procedures)`
  - `getSignUrl(...)`
  - 输出对象里同时保留 `p`、`s`、`ss`、`gs`
- 适合借鉴：
  - 多个样品合并成一份采样记录
  - 报告头取自流程表单，明细取自样品数组
  - 需要按签字人组合拆分报告
- 图像级关系：
  - 单张模板页中的一行表格原型，会在结果图里展开成多行并跨 3 页。
  - 这组样本证明“一个模板页 + 一个数组字段”完全可能生成多页结果。

## 092α、β表面污染测量结果处理表

- 返回根：单个 `reports` 对象
- 主过滤：
  - `sample.tempId`
  - `gauging.gaugingTemplateName === '006α、β表面污染测量记录表'`
- 关键写法：
  - `arr1` / `arr2` 双明细表
  - 结果按 `t_num` 排序
- 模板线索：
  - 签名图片在模板里直接写 `=headerUrl + t_shuMan[0].val`
- 适合借鉴：
  - 一个报告对象里挂多个 `<data>` 表
  - 同一批样品展开成两个不同视角的结果表
- 图像级关系：
  - 同一张模板页里包含两个表格数据区。
  - 结果图会自动分页成续表。
  - 这组样本还能看到字段类型和模板写法不匹配时的异常表现，例如对象直接显示为文本信号。

## 099中子辐射剂量率测量结果处理表

- 返回根：`outputData` 数组
- 主过滤：
  - `sample.tempId`
  - `gauging.t_version === '024'`
- 关键写法：
  - `partData` / `gaugingData`
  - `ObjAssign` / `getValue`
  - 先取一条样品+一条 gauging 组公共头，再喂 `gaugingList`
  - 可选补空行逻辑
- 适合借鉴：
  - 一份公共头 + 一个明细循环表
  - 需要把签名数组自动转图片 URL
  - 需要时间字段裁剪到 `HH:mm`
- 图像级关系：
  - 一个模板页被重复渲染成两页。
  - 第一页放满循环行，第二页承接剩余行和空白补位。

## 137流出物样品γ谱分析原始记录表

- 返回根：`outputData` 数组
- 主过滤：`gauging.gaugingTemplateName === '137流出物样品γ谱分析原始记录表'`
- 主分组：每个样品一份
- 关键写法：
  - 先按样品创建输出壳
  - 再把同样品的 `gaugingList` 追加进去
- 适合借鉴：
  - 同一项目下每个样品单独出一份分析原始记录
  - 一个样品对应多个因子明细行
- 图像级关系：
  - 两张模板图里，一张更像说明版，一张更像实际模板版。
  - 结果图是“每个样品一页”，而不是“一个总表多页”。

## X-γ辐射剂量率测量记录表

- 返回根：`outputData` 数组
- 主过滤：`gauging.t_version === '005'`
- 主分组：按 `t_changSuo` 分组
- 关键写法：
  - 第一轮扫描先建公共头
  - 第二轮扫描再往对应组里塞 `gaugingList`
- 适合借鉴：
  - 先按地点/区域聚合，再生成每个地点一份报告
  - 同一组下多个样品共用同一页头
- 图像级关系：
  - 模板中的每个点位行内部实际上是“双层测量值子行”。
  - 结果图能帮助确认脚本不是简单一行一条，而是一个点位对象包含两排读数。

## 原子吸收光谱测定水样中元素分析原始记录表

- 返回根：`outputData` 数组
- 主过滤：`gauging.gaugingTemplateName === '062原子吸收光谱测定水样中元素分析记录表'`
- 关键写法：
  - `t_glQX.fkCases[0].t_qx`
  - 由关联曲线数据组装 `newData`
  - 收集所有因子名后合并成 `factorNames`
  - 固定 6 行补空对象
- 适合借鉴：
  - 明细表之外，还要从关联档案里再构建一张辅助校准表
  - 报告头和曲线表来自不同层的数据
- 图像级关系：
  - 第 1 张模板页负责方法/曲线概览。
  - 第 2 张模板页负责样品结果循环表。
  - 结果图显示后半部分是同一页模板的连续分页，而不是多个完全不同模板。

## 氡析出率测量结果处理表

- 返回根：`outputData` 数组
- 主过滤：`gauging.gaugingTemplateName === '113、007氡析出率测量记录表'`
- 关键写法：
  - `commonData.t_htName = form.t_htName`
  - `commonData.t_wtName = form.t_wtName`
  - 固定 4 行补空对象
- 适合借鉴：
  - 表头字段要强行以 `form` 为准，不能完全依赖 `ObjAssign`
  - 单一检测项目家族的固定高度结果表
- 图像级关系：
  - 一张模板页被重复渲染成多张续表页。
  - 这组样本非常适合学习“同页模板反复渲染 + 自动页眉续表标题”的模式。

  - 第 1 张模板页偏向曲线和方法信息。
  - 第 2 张模板页是样品结果行循环表。
  - 结果图基本是固定两页结构。

## 监测报告（电磁）

- 返回根：`formJs`
- 主过滤：
  - `factorName` 在 `工频电场/工频磁场`、`环境噪声`
  - `t_version` 在 `003`、`022`
- 主分组：先按点位采集目标检测项，再按 `t_TestDate + factorName` 聚合环境条件
- 关键写法：
  - 直接在 `formJs` 上补 `monitoringItemSubTitle`
  - 构造 `gpdcList`、`hjzsList`、`jctjList2`
  - `gpdcList` / `hjzsList` 行对象优先保留“当前 `gauging` + 点位名 `t_dianweiM` + `sortNum`”，不要把整份 `form` 合并进每行
  - 先从 `caseName.slice(-3)` 生成 `sortNum`，再对明细行排序
  - `gpdcList` / `hjzsList` 入表前还要校验 `t_nameYQ`、`t_modeXH`、`t_numYQ`
  - `jctjList2` 不是简单取首行，而是把 `jctjList1` 按 `t_TestDate + factorName` 分组后，再把 `t_wendu`、`t_shidu` 聚成区间文本
  - 模板中大量内联 JS 控制标题、签名、日期、图片
- 适合借鉴：
  - 多页监测报告，但脚本只返回一份总对象
  - 大量版式文案由模板内联 JS 决定
- 图像级关系：
  - 模板 4 张图对应一份完整 4 页报告。
  - 第 2 页几乎全静态。
  - 第 3 页是否出现 1 个还是 2 个监测项目表，依赖脚本提供的 `monitoringItemSubTitle` 和列表数组。
  - 第 4 页的图片和三处签名日期都依赖模板内联 JS 联动脚本字段。

## 使用这些案例类型的顺序建议

1. 先选一个“输出结构最像”的样本。
2. 再选一个“数据源最像”的样本。
3. 如果还有图片/签名/标题拼接问题，再补看占位符说明最接近的那个样本。

不要一次把所有案例类型都混成一锅“统一规范”。这份 skill 更适合按家族复用，而不是按教科书重写。
